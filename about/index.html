How the pieces fit together

Each folder is a real page: index.html for direct loads, main.json for the SPA router to fetch, main.js for behavior. This clock proves mount/unmount cleanup runs either way.

  • router.js — optional, drop-in SPA engine, zero app deps
  • main.json — { html, title } the router injects into #view
  • main.js — exports mount(root), optionally returns an unmount fn
  • No router.js tag → link clicks do full page loads, still works
